# 需求-frida可视化操作和管理-v2.md

## 需要做的GUI问题修复

启动选中应用，没有启动ANDROID APP。



**启动Android App的命令：**`frida -H 127.0.0.1:[主机上监听的端口号] -l [脚本路径] -f [android app的包名，如：com.android.chrome]`



**frida的`-l`参数用法：**如果没有绑定脚本则不要指定这个参数。如果绑定多个脚本则要这么写：`-l [脚本路径1] -l [脚本路径2] -l [脚本路径3]`。



## 需要做的GUI优化

### 应用列表

PID、应用名、包名，我期望能够选中其一进行复制。



### 按钮点击后的交互

点击Kill选中进程、启动选中进程，执行成功后立即刷新应用列表。如果失败则需要弹出错误信息。



### 应用启动优化

1. 应用列表页每一项后面都添加一个配置脚本的按钮，点击后允许添加脚本路径。
2. 点击“启动选中进程”按钮，直接启动APP，启动的时候带上2.a配置的脚本。列表页点击某一项，然后直接回车，运行“启动选中进程”按钮的功能。
4. 存储应用和脚本绑定关系：新增一个数据库存储这些数据。



#### 配置脚本的按钮

点击后打开一个对话框显示一个列表，显示当前绑定的脚本、删除绑定的脚本、还能够添加已经绑定的脚本，列表不分页，超过高度的部分滚动显示出来。



**列表数据查询**：查询`frida_device_app_script_bind`表，根据`device_type + app_identity`查询



**列表要显示的内容：**

1. 序号：从1开始
2. 添加时间：本地机器时区的时间，格式：年-月-日 时:分:秒。
3. 脚本名：如果过长单元格显示不下的情况，通过横向滑动能够看到后面的内容。允许选中复制。
4. 脚本路径：如果过长单元格显示不下的情况，通过横向滑动能够看到后面的内容。允许选中复制。

按照添加时间倒排。



**删除脚本绑定：**选中列表中的一项点击“删除按钮”。



**添加脚本绑定：**直接在对话框中添加一个输入框和一个添加按钮。输入框有内容的情况下启用添加按钮，否则禁用。在输入框中按回车或点击添加按钮，对输入框内容trim后执行添加逻辑。



**添加逻辑：**

1. 对脚本路径trim，如果trim后路径是空，则弹出路径不能为空，然后终止执行。
2. 根据`device_type + app_identity + script_path`查询是否已经添加过：
   1. 如果添加过：提示脚本已绑定，不要重复添加。
   2. 如果没有添加过：在数据库中插入数据。
3. 添加成功后刷新脚本绑定列表。

## 数据库

### 使用的数据库

sqlite



### 存放位置

`~/bw-frida/frida-server/frida.db`



### 数据库表通用字段

**所有数据库表都要有下列字段：**

| 列          | 类型               | 描述                       |
| ----------- | ------------------ | -------------------------- |
| id          | unsinged long      | 非空。自增主键。           |
| create_time | 毫秒级别的日期类型 | 非空。创建时间，毫秒级别。 |
| modify_time | 毫秒级别的日期类型 | 非空。修改时间，毫秒级别。 |



### 表：frida_device_app_script_bind

| 列           | 类型          | 描述                                                   |
| ------------ | ------------- | ------------------------------------------------------ |
| device_type  | varchar(64)   | 非空。设备类型。枚举值：android、ios、windows、linux。 |
| device_id    | varchar(64)   | 非空。设备ID。对于android来说设备ID可以从adb。         |
| app_identity | varchar(256)  | 非空。app身份。对于android来说就是app的包名。          |
| script_path  | varchar(1024) | 非空。绑定的脚本的路径。                               |

每一个脚本绑定的信息就是一条数据。



**普通索引：**

1. device_id + app_identity
2. app_identity
